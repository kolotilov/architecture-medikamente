## 1. Классы конфиденциальных данных

### Персональные данные (PII)
- ФИО
- Дата рождения
- Телефон, e‑mail
- Адрес
- Место работы/учёбы
- Паспортные данные

### Медицинские данные (PHI)
- История болезни
- Диагнозы
- Назначения
- Результаты анализов
- Рецепты

### Финансовые данные
- История платежей
- Банковские реквизиты
- Номера банковских карт

### Данные сотрудников
- Учетные записи
- Права доступа
- Кадровые и зарплатные данные
- Логи аутентификации

---

## 2. Аудит текущего состояния

### 2.1 Соответствие требованиям

| Процесс                         | Требование безопасности                            | Текущее состояние     |
|--------------------------------|----------------------------------------------------|------------------------|
| Хранение медицинских данных    | Шифрование, RBAC, аудит                            | Отсутствует            |
| Передача данных в 1С           | TLS, VPN, защищённый API                           | Отсутствует            |
| Доступ к Excel-файлам          | Контроль прав, аудит действий                      | Не настроено           |
| Ввод данных вручную            | Автоматизация, снижение риска утечек               | Отсутствует            |
| Классификация и тегирование    | Data tagging, классификация уровней конфиденциальности | Отсутствует       |

### 2.2 Проблемные зоны

- Нет шифрования данных ни при хранении, ни при передаче
- Данные хранятся в Excel/сканах на общих дисках
- Нет разграничения доступа по ролям
- Нет логирования и мониторинга
- Нет классификации и тегов на уровне данных
- Ввод и передача данных вручную

## 3. Рекомендации по улучшениям

### 3.1 Методы защиты данных

| Тип данных       | Метод защиты                                           |
|------------------|--------------------------------------------------------|
| PII              | Шифрование (AES-256), маскирование, контроль доступа  |
| PHI              | Шифрование, псевдонимизация, RBAC                      |
| Финансовые       | Токенизация, шифрование, маскирование                  |
| Сотрудники       | Шифрование, контроль доступа, аудит                    |

### 3.2 Тегирование и классификация

**Категории тегов:**
- Уровень конфиденциальности: `INTERNAL`, `CONFIDENTIAL`, `RESTRICTED`
- Тип данных: `PII`, `PHI`, `FINANCIAL`, `OPERATIONAL`
- Доступ: `MEDICAL_STAFF`, `ACCOUNTING`, `ADMIN_ONLY`
- Срок хранения: `SHORT_TERM`, `LONG_TERM`

**Инструменты:**
- Apache Atlas
- Microsoft Purview
- Подключение к хранилищу (Data Lake, S3)

### 3.3 Контроль доступа и аудит

- Внедрение RBAC/ABAC
- Open Policy Agent / Keycloak
- SIEM для аудита действий (например, Wazuh)
- DLP для контроля утечек
- VPN и TLS между системами

- Микросервисный подход
- API-шлюзы с авторизацией
